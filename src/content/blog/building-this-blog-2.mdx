## Reading progress bar

A fixed 2px bar at the top of the viewport fills left-to-right as you scroll. It uses `transform: scaleX()` rather than `width` to avoid layout recalculation on every scroll event:

```tsx title="src/components/ReadingProgress.tsx"
export function ReadingProgress() {
  const [progress, setProgress] = useState(0);

  useEffect(() => {
    const update = () => {
      const scrolled = window.scrollY;
      const total = document.documentElement.scrollHeight - window.innerHeight;
      setProgress(total > 0 ? scrolled / total : 0);
    };
    window.addEventListener("scroll", update, { passive: true });
    return () => window.removeEventListener("scroll", update);
  }, []);

  return (
    <div
      style={{
        position: "fixed",
        top: 0,
        left: 0,
        right: 0,
        height: "2px",
        background: "var(--primary)",
        transformOrigin: "left",
        transform: `scaleX(${progress})`,
      }}
    />
  );
}
```

`passive: true` on the scroll listener tells the browser the handler won't call `preventDefault()`, allowing it to scroll without waiting for JavaScript. The animation stays smooth even on slower devices.

## Table of Contents

The TOC is generated automatically from the post's headings. Astro passes a `headings` prop to MDX layouts — an array of `{ depth, slug, text }` objects. The component filters to H2 and H3:

```astro title="src/components/TableOfContents.astro"
const toc = headings.filter((h) => h.depth === 2 || h.depth === 3);
```

It uses an `IntersectionObserver` to highlight the currently visible section:

```js
const observer = new IntersectionObserver(
  (entries) => {
    for (const entry of entries) {
      if (entry.isIntersecting) {
        setActive(entry.target.id);
      }
    }
  },
  { rootMargin: "0px 0px -80% 0px" },
);
```

The `rootMargin` makes the observer fire when a heading enters the top 20% of the viewport — so the TOC updates before the section fills the screen.

<Callout type="note">
  The TOC only appears when a post has 3 or more headings. This post has enough,
  so you can see it above.
</Callout>

## Callout blocks

Four semantic types, each with a colored left border and icon:

<Callout type="note" title="Note">
  Supplementary context that's useful but not critical to the main point.
</Callout>

<Callout type="tip" title="Tip">
  A practical shortcut or technique the reader might appreciate.
</Callout>

<Callout type="warning" title="Warning">
  Something to be careful about before proceeding.
</Callout>

<Callout type="danger" title="Danger">
  This action is irreversible or destructive.
</Callout>

The Callout component is an Astro component with a `<slot />` for the body — it renders at build time with zero client JavaScript.

## Math with KaTeX

Math formulas render via KaTeX, which runs at build time through `remark-math` + `rehype-katex`. No JavaScript ships to the client for math rendering.

Inline formulas: the famous mass-energy equivalence is $E = mc^2$.

Block formulas are centered on their own line:

$$
\nabla \cdot \mathbf{E} = \frac{\rho}{\varepsilon_0}
$$

$$
\sum_{k=1}^{n} k = \frac{n(n+1)}{2}
$$

The KaTeX stylesheet is loaded from a CDN link in `<head>` — a single HTTP request for styles, no JavaScript bundle cost.

## Keyboard shortcut styling

`<kbd>` elements render with a raised border that mimics a physical key:

Press <kbd>Ctrl</kbd> + <kbd>K</kbd> to open search. Press <kbd>Escape</kbd> to close it.

The CSS adds a heavier bottom border to create the depth effect:

```css
kbd {
  display: inline-block;
  padding: 0.15em 0.4em;
  border: 1px solid var(--border);
  border-bottom-width: 2px;
  border-radius: 0.3rem;
  background: var(--muted);
}
```

## Heading anchors

Every heading has a permanent `#` anchor link. Hover over any heading in this post and you'll see it. Click to copy a direct link to that section.

The anchors are added by `rehype-autolink-headings` at build time. The CSS shows the `#` only on hover:

```css
.prose :is(h1, h2, h3, h4) > a::before {
  content: "#";
  opacity: 0;
  color: var(--primary);
  transition: opacity 150ms ease;
}
.prose :is(h1, h2, h3, h4):hover > a::before {
  opacity: 0.6;
}
```
