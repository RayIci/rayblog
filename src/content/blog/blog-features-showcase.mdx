## Callout Boxes

Use `<Callout>` with `type="note|tip|warning|danger"` and an optional `title` prop.

<Callout type="note" title="A note">
  Use this for supplementary context that doesn't interrupt the main flow.
</Callout>

<Callout type="tip" title="Pro tip">
  Tips highlight best practices or shortcuts worth knowing.
</Callout>

<Callout type="warning" title="Heads up">
  Warnings flag things that could cause confusion or subtle bugs.
</Callout>

<Callout type="danger" title="Danger zone">
  Use this for destructive, irreversible, or security-critical actions.
</Callout>

## Code Blocks

Powered by [Expressive Code](https://expressive-code.com) with Catppuccin themes, line numbers, and titles.

```ts title="src/lib/utils.ts"
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

### Diff highlighting

Mark added/removed lines with `ins` and `del` annotations:

```ts title="example.ts" del={2} ins={3-4}
function greet(name: string) {
  return `Hello, ${name}`;
  const greeting = `Hello, ${name}!`;
  return greeting;
}
```

## Series Support

Posts can be grouped into a series using the `series` and `seriesOrder` frontmatter fields:

```yaml
series: "Building RayBlog"
seriesOrder: 1
```

A numbered navigation block appears at the top of the post, linking all parts together.
