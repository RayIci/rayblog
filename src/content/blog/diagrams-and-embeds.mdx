This post demonstrates the rich media components available in this blog: Mermaid diagrams for technical illustrations, YouTube embeds for video content, and Tweet embeds for social media references.

## Mermaid diagrams

[Mermaid](https://mermaid.js.org/) lets you write diagrams as code directly in your post. Diagrams are rendered in the browser and respect the current light/dark theme.

## YouTube embeds

YouTube videos embed responsively using the privacy-respecting `youtube-nocookie.com` domain, which doesn't set tracking cookies until the user clicks play.

<YouTube id="dQw4w9WgXcQ" title="Rick Astley — Never Gonna Give You Up" />

Usage in MDX:

```mdx
import YouTube from "@/components/YouTube.astro";

<YouTube id="dQw4w9WgXcQ" title="Video title for accessibility" />
```

## Tweet embeds

Tweets are fetched at build time using Twitter's oEmbed API, so there's no client-side JavaScript loading Twitter's widget bundle on page load. The widget script is only added when a tweet actually appears on the page.

<Tweet id="20" />

<Callout type="note">
  If the oEmbed fetch fails (e.g., the tweet was deleted or the API is
  unavailable), the component falls back to a plain link to the tweet.
</Callout>

Usage in MDX:

```mdx
import Tweet from "@/components/Tweet.astro";

<Tweet id="1234567890" />
```

## Image zoom

Any image in a post can be clicked to zoom in. The zoom overlay uses the site's background color and respects dark mode. To opt out of zoom for a specific image, add the `no-zoom` class.

## Image zoom

Click any image to open it in a full-screen overlay. The overlay respects dark mode and closes on click, scroll, or <kbd>Escape</kbd>.

![A mountain lake at dusk](https://picsum.photos/seed/rayblog-lake/800/450)

![An aerial city view](https://picsum.photos/seed/rayblog-city/800/450)

The zoom is powered by [medium-zoom](https://github.com/francoischalifour/medium-zoom), initialized in the blog layout targeting all prose images. Add `class="no-zoom"` to an image to opt it out.

## Combining rich media

These components compose naturally with the rest of the prose. You can drop a diagram in the middle of an explanation, follow it with a callout, and continue the text — the layout adapts automatically.

The goal is that adding rich media should feel like a natural extension of writing, not a context switch into some embed configuration UI.
